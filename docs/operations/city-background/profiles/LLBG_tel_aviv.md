## 1) 结论摘要（历史 METAR + 站点背景）

1. 地中海海岸平原城郊机场，既不能当 Tel Aviv 海边市区热岛，也不能当纯内陆站处理。
2. 湿凉季(11-3)偏暖风向 `E(22.6°C) / SE(20.7°C)`；偏冷风向 `NW(20.2°C) / S(18.2°C)`；温差约 `2.4°C`；峰值时刻约 `13:15` vs `13:30`。
3. 峰值偏早，后段常见的是锁温或回落，而不是持续上冲。
4. 降水重置日占比不低，午前启动迟滞和地面湿润残留要单独处理。
5. 风向切换敏感，主导风向组合为 NW:19%, W:16%, SE:14%，转向常常比当前温度更早发出路径变化信号。
6. 同月最热窗口通常出现在 Aug，对应月度 Tmax 中位约 33.0°C。

## 2) L2 核心识别框架

- 核心定位：地中海海岸平原城郊机场，既不能当 Tel Aviv 海边市区热岛，也不能当纯内陆站处理。
- 西到西北风海风推进会明显压缩午后上沿；东到东南偏陆风背景更容易保留更高上限。
- 站点西侧受地中海影响，东侧又连接向内陆和丘陵过渡，风向切换常比单小时温度更早提示路径变化。
- 春秋转季的海风切入时点很关键；盛夏则更要看海风是否提前锁死午后增温。

## 12) 旧版易误判场景

1. 把 LLBG 当成 Tel Aviv 海边市区代理，会低估偏陆风日的升温潜力。
2. 反过来按纯内陆热站外推，又会低估海风回灌和湿层对尾段温顶的截断。
3. 若不单独处理雨后重置日，暖季上限会被系统性高估。

## 13) 实况优先观察顺序

1. 先看西北侧海风是否推进，再看露点和低云/湿层，最后判断峰值窗内是否仍保有偏陆风增温空间。
2. 风向切换应视为 live pivot，必要时比同小时模式偏差更值得信任。
3. 有降水或地面湿润残留时，优先防高估；除非午前快速放晴且斜率恢复。
4. Apr 常见干混合上冲，晴空低云少的日子要防旧版保守低估。
