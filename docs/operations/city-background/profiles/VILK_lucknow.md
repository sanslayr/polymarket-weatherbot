## 1) 结论摘要（历史 METAR + 站点背景）

1. 恒河平原站点，晨间低能见/湿层散除时点与对流季触发风险，往往比单小时气温更决定最高温。
2. 雨季 / 季风季(6-9)偏暖风向 `NW(37.3°C) / W(35.0°C)`；偏冷风向 `E(34.3°C) / SE(33.4°C)`；温差约 `2.3°C`；峰值时刻约 `14:30` vs `14:30`。
3. 峰值偏早，后段常见的是锁温或回落，而不是持续上冲。
4. 湿热滞留占比不低，露点背景会显著改写升温效率，不能只看气温斜率。
5. 降水重置日占比不低，午前启动迟滞和地面湿润残留要单独处理。
6. 风向切换敏感，主导风向组合为 W:33%, E:31%, NW:21%，转向常常比当前温度更早发出路径变化信号。

## 13) 旧版易误判场景

1. 把 Lucknow 全年都按干热内陆站处理，会低估 haze/fog 与对流季截断的影响。
2. 只盯温度斜率而忽视能见度和露点差变化，容易错判是否已进入真正混合层增长。
3. 若不单独处理雨后重置日，暖季上限会被系统性高估。
4. 晨间低能见/湿层较重时，启动偏慢更常见，不宜直接套 clean-ramp 日模板。

## 14) 实况优先观察顺序

1. 先看 BR/HZ/FG 与能见度改善时点，再看露点差是否扩张，最后看午后对流触发风险是否切断峰值窗。
2. 露点和湿层厚度的解释优先级要提高，单看温度斜率容易误判。
3. 风向切换应视为 live pivot，必要时比同小时模式偏差更值得信任。
4. 有降水或地面湿润残留时，优先防高估；除非午前快速放晴且斜率恢复。
5. Apr 常见干混合上冲，晴空低云少的日子要防旧版保守低估。
