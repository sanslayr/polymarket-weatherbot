# P2/P3 Recognition Review (Ankara + London) — 2026-03-01

## Scope
Review focused on P2/P3 detector readiness and framework reuse:
- frontal / boundary-layer / foehn-proxy family
- baroclinic scaling
- sounding readiness
- anti-site-patch architecture consistency

## Dataset state used
- London (EGLC): richer slice coverage in current cache window (ecmwf bundle had 5 slices)
- Ankara (LTAC): partial coverage in current cache window (ecmwf bundle had 2 slices)
- 3D provider default strategy: `gfs-grib2`
- hourly forecast default strategy: `openmeteo`

## Key findings

### 1) Topography data feasibility (for foehn/channel-wind)
- **Available now**: station elevation can be queried via Open-Meteo elevation API.
- **For robust foehn/channel-wind** we still need local DEM context (upwind/downwind relief, ridge orientation, valley axis).
- This is static/semi-static data and can be cached long-term per station (good fit for reusable framework).

### 2) Baroclinic score calibration
- Previous `coupling_score = g_p * g_t` produced near-zero tiny values and weak interpretability.
- Updated to robust-normalized score:
  - normalize pressure-gradient and thermal-gradient by p90
  - combine via sqrt product
  - scale/clamp to [0,1]
- Output now includes:
  - `coupling_score`
  - `pressure_gradient_norm`
  - `thermal_gradient_norm`

### 3) Sounding (P3) readiness
- Current decisions show `thermo.has_profile=false` and `quality=missing_profile`.
- Meaning: framework is CAPE/CIN-ready, but raw profile ingestion/solver still not fully connected.

## Framework-level changes completed in this round

### A) Provider-aware cache (no hidden cross-provider collisions)
- forecast decision cache key now includes `synoptic_provider`
- 3D bundle metadata now stores `synoptic_provider`
- fallback bundle lookup now filters by provider
- effect: avoids stale/degraded result contamination when switching openmeteo/gfs 3D paths

### B) Main-object ranking no longer distance-only
- Added reusable ranking composition:
  - type weight (baroclinic/frontal/dynamic/advection/shear/...)
  - confidence weight
  - impact-scope weight
  - coherence/surface-coupling contribution
  - distance penalty
- sorted by `rank_score` first, then distance
- effect: reduces generic-nearest object dominating semantic main system

### C) Reusable (non-station-patch) orientation
- No Ankara/London hardcoded branch in ranking/scoring
- logic remains generic across stations; station specifics should live in data/config layers (future topography profile store)

## Remaining gaps
1. Foehn/channel-wind detector still pending DEM-context ingestion.
2. Sounding raw profile solver (CAPE/CIN/LCL/LFC/EL full chain) still pending.
3. Threshold adaptation by regional climatology still pending (current thresholds are global/default).

## Recommended next steps
1. Add station DEM-context cache (`station_topography_context`) and implement generic foehn/channel proxy.
2. Add sounding raw profile ingestion + thermo solver; keep full/lite/skip contract.
3. Add calibration harness over multi-station historical windows for threshold tuning.
